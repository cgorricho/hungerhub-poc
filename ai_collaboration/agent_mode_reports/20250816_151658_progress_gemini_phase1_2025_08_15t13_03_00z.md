Overview
- Phase 1 delivers the initial Dash application refactor, logging standardization, ETL updates, and test suite stabilization aligned with the Techbridge AI Collaboration Framework.
- Simulation/demo fallback paths have been removed to enforce real-data-only flows. Empty states are surfaced clearly in the UI instead of auto-simulating data.
- CI runs pytest with ruff and mypy in non-blocking mode; we can make linting/typing checks blocking in a follow-up as desired.

What changed in Phase 1
1) Dash app (src/dashboard/dash/enhanced_app.py)
   - Uses standardized data and logs directories via the shared paths utility (no ad-hoc project_root/path construction).
   - Removes simulation/demo fallback logic (e.g., geographic weight map, donor fallbacks). App now fails fast or shows clear empty-state messaging.
   - Replaces print statements with logging or silent fail-fast handling where appropriate.

2) Logging configuration (Dash and Streamlit dashboards)
   - Ensures logs directory creation via the paths utility.
   - Respects LOG_LEVEL and LOG_DIR environment variables.
   - Uses rotating file handlers with timestamped filenames in the configured logs directory.

3) ETL scripts
   - create_unified_real_data.py: adopts logging and the shared paths utility for data/logs directories.
   - full_data_extractor.py: reads Oracle client path from ORACLE_CLIENT_LIB; aligns CHOICE_* and ORACLE_* environment variables for DB connectivity precedence.

4) Tests and enforcement of real-data-only policy
   - tests/dashboard/test_metrics_shapes.py: index name set to "flow_stage" for funnel chart expectations.
   - tests/logging/test_logging_config.py: assertions updated to the project logger expected INFO level.
   - tests/test_unified_shapes.py: skips when parquet test data is absent (avoids hard failures unrelated to code).
   - Removed lingering simulation artifacts and demo fallbacks (e.g., donor strings), aligning with the no-simulation policy.

5) CI
   - GitHub Actions workflow executes pytest. Ruff and mypy run in a non-blocking mode.
   - We can phase in blocking lint/type checks after review.

Notes and constraints
- Oracle integration: Full sequential test is skipped if Oracle environment variables are not configured. This prevents CI failures when secure environment details are unavailable.
- Data availability: The dashboard surfaces empty-state messaging when expected real datasets are missing; it no longer simulates.
- Logging scope: Project logger is configured at INFO by default; root logger behavior is not overridden unless explicitly configured by the environment.

How to run (high level)
- Ensure required environment variables are set (notably Oracle-related: ORACLE_CLIENT_LIB, CHOICE_*/ORACLE_* credentials as applicable).

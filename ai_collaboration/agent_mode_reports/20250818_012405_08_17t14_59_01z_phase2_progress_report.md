## Summary
Phase 2 focuses on externalizing the Oracle table catalog configuration and aligning the production extractor to consume this config with safe fallbacks, while ensuring CI visibility and reliability. ## Scope & Objectives
- Externalize table catalog into a versioned, documented JSON file under config/.
- Refactor full_data_extractor to load table lists from the external config with validation and fallback to built-in defaults.
- Ensure CI workflow is detected and can be triggered on push/PR and manually.
- Provide minimal test alignment and skip strategy for Oracle integration when env is absent, to keep CI green.

Out of scope for Phase 2:
- Actual Oracle extraction runs (require secure env). Those remain manual/Phase 3.
- Broader ETL pipeline refactors or modeling changes beyond the catalog source-of-truth.

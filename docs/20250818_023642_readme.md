# HungerHub Analytics POC - 2-Week Implementation

## 🎯 Project Overview
**Objective**: Create an Oracle → Python → Plotly Dash analytics pipeline for HungerHub data analysis  
**Duration**: 14-day accelerated POC  
**Framework**: Techbridge AI Collaboration (Agent Mode + Gemini CLI + Developer)  

## 📊 Deliverables
- **3-Page Interactive Dashboard**:
  1. Executive Summary (KPIs + trends)
  2. Donation Analytics (donors + geographic)
  3. Agency Operations (orders + fulfillment)
- **ETL Pipeline**: Oracle extraction, transformation, Parquet storage
- **Analytics Engine**: Core KPIs and trend analysis
- **Azure Deployment**: Production-ready dashboard

## 🏗️ Technical Stack
- **Database**: Oracle (cx_Oracle connector)
- **Backend**: Python (pandas, SQLAlchemy)
- **Frontend**: Plotly Dash + Bootstrap components
- **Storage**: Parquet files for fast I/O
- **Deployment**: Azure VM + nginx reverse proxy

## 📦 Git LFS Setup (for data files)
Large data artifacts (Parquet/CSV) under data/ are tracked via Git Large File Storage (LFS).

Setup (one-time per machine):

```bash
# Install Git LFS (see https://git-lfs.com/ for OS-specific instructions)
# Ubuntu example:
sudo apt-get update && sudo apt-get install -y git-lfs

# Initialize in your local repo
git lfs install

# Patterns are preconfigured in .gitattributes, but you can verify:
git lfs track "data/**/*.parquet" "data/**/*.csv"

# Commit hooks are automatic; just pull normally and LFS will fetch pointers
```

Notes:
- Only data/**/*.parquet and data/**/*.csv are tracked with LFS; source code remains in standard Git.
- If you add new large data formats, update .gitattributes accordingly and run git lfs track for them.
